    我的第一篇 先不废话 教你Hexo

    发表于 2022-01-12 更新于 2022-01-17 分类于 hexo

    哈喽大家好，我是苏家树，这是我的第一篇GitHub page(可以当作一个私人blog吧)，这里就先不自我介绍了，先教大家怎么搭自己的blog吧！ 我也是跟着别人一点一点学的，在这里分享一下，0基础的朋友也不是问题！

    OK 那我们就开始把GitHub page fuck! ….. 不好意思不是这个意思 把 GitHub page 拿下！

    废话不多说

    开车！

    不是不是!

    开学!

    开始搭建

    安装 node
    我会把需要在terminal输入的指令用橙色标记

    因为 hexo(我们要学的的) 是基于 node 框架的，所以呢，我们首先需要下载安装node，下载地址：nodejs
    安装完之后，我们打开terminal，输入 node -v，如果返回下图所示，那么就说明你安装 node 成功了。

    初始化blog
    1. 安装成功后，我们在terminal运行如下命令来安装hexo：npm install hexo-cli -g
    2. 初始化博客目录hexo init user.github.io (这里的user换成你自己的user)
    3. 初始化完成后，在terminal进入我们的目录，输入：cd user.github.io
    4. 安装，接着输入：npm install
    5. clean一下：hexo clean
    6. 然后生成静态页面： hexo g

      g 就是generate, 生成的意思

    7. 接着把你的网站运行起来，输入： hexo s

      s 就是server ,在服务器运行的意思

    打开你的浏览器，输入 localhost:4000。 自此，你的个人网站就这么速度的搭建起来了！

    • 到这一步出现问题的小伙伴加我v 我帮你解决 j0306s_1114.

    接下来优化 hexo 博客主题

    修改你的网站中的个人信息， 进入你的网站目录，网站目录电脑已经自动帮你建好了，在电脑里搜索你的user.github.io就能找到，是一个文件夹， 打开 _config 文件，这个文件是用来配置你的网站信息的。

    这里列出我目前这个page的简单配置，具体可以看我的这个Hexo配置文档

    title: 牛逼的page
    subtitle: 牛逼的page，自然来自牛逼的人
    Description: 一个热衷于帮别人变牛逼的小lj
    keywords: 牛逼page, jiashu
    author: 微信：j0306s_1114 有问题可以私信我
    language: zh-CN

    选一个Hexo主题

    刚搭建完的网站，是不是觉得有点丑，不符合你的漂亮的脸庞？ 反正我是这么觉得的，那么就选一个主题来装饰一下吧，在这里可以选择你要的主题：https://hexo.io/themes/

    下载Hexo主题

    选择好了你的主题之后，就可以下载主题资源了，比如我这里选择一个 NEXT 主题来演示一下,在你的目录中输入以下命令：

    git clone https://github.com/next-theme/hexo-theme-next.git themes/next

    上面这个链接要到这github里面复制

    配置主题

    主题下载完之后，在你根目录下的 _config.yml 文件中，修改 theme 为你的主题名字：

    theme: next

    重新生成和运行

    hexo g
    hexo s

    再重新打开terminal，hexo g, s访问一下看看，是不是比之前好看多了

    让全世界的人都认识你：

    当然最重要的一步，就是公开啦，谁不想把这玩意拿出去装个逼呢， 使用GitHub Pages，使用它就可以部署自己的网站啦。

    创建 Github pages 仓库

    接着创建一个 xxx.github.io 的 public 仓库，这里的xxx也必须是你的名字，要跟刚才那个user一样， 比如我写的是 jiashusu.github.io,那么到时我就可以通过 jiashusu.github.io 来访问我的网站了。


    创建完成之后，那么你就有自己的 Git 地址了

    安装 hexo-deployer-git

    terminal cd到你的博客目录下：

    1. 先在terminal 输入cd 然后一个空格 把你blog文件夹拖到terminal里面 意思就是告诉terminal要在这个文件夹搞事情了

    然后输入如下命令，这样你在本地写的文章才能 push 到 GitHub 上面去。

    npm install hexo-deployer-git --save

    ### 配置你的Git
    打开你的配置文件(config)，找到deploy， 然后输入你的 git 地址:
    >
    type: git
    repo: https://github.com/xxx/xxxx.github.io.git

    上面删掉的行不通了现在，GitHub更新了，需要密钥
    我来教大家一下密钥设置

    生成新 SSH 密钥

    重新打开 Terminal（终端）Terminal（终端）Git Bash。

    粘贴下面的文本（“”里面的内容 替换为你的电子邮件地址）。

    ssh-keygen -t rsa -C "geovbox@163.com"

    这将以提供的电子邮件地址为标签创建新 SSH 密钥。

    > Generating public/private algorithm key pair.

      提示您“Enter a file in which to save the key（输入要保存密钥的文件）”时，按 Return 键。 这将接受默认文件位置。

      > Enter a file in which to save the key (/Users/you/.ssh/id_algorithm): [Press enter]
      > Enter a file in which to save the key (/c/Users/you/.ssh/id_algorithm):[Press enter]
      > Enter a file in which to save the key (/home/you/.ssh/algorithm): [Press enter]

    1. 在提示时输入安全密码，写一个好记一点的就行 我的是我生日0503

      > Enter passphrase (empty for no passphrase): [Type a passphrase]
> Enter same passphrase again: [Type passphrase again]

      出现下面这种就表示注册成功了

      复制你的git pub：

      cat ~/.ssh/id_rsa.pub

      然后废话不多说 直接上图教你

      最后一步！！！

      复制你仓库的SSH码

      到config配置文件里找到deploy

      type: git
      repository: 你刚刚复制的那个git
      branch: master

      推送你的网站到Github上

      • 重新打开terminal
        • 先clean一下 ：clean
        • 生成public ：hexo g
        • 直接输入命令 ：hexo d

      d 就是 deploy ， 部署上去的意思。

      访问你的网站

      push上去之后，你就可以直接在浏览器输入你的 xxx.github.io 就可以访问啦！
