我会推出一篇全面讲解git的新篇，今天在这里先简单说一下怎么注册多个git吧：
因为你的GitHub page库名字必须和你的GitHub username一样，这导致一个github账号只能拥有一个GitHub page，如果你想多个page，比如我就有另一个page：休斯顿相亲网： houstonlove.github.io 或者放张这个更详细点：

8848 钛金手机，双系统，一个婚内，一个婚外， 一个工作，一个生活的话，你就需要多个page，创建GitHub账号容易，但是每个账号需要不同的git密钥，就一个大大的漏，那我们就来学怎么分配git密钥，在你需要更新小三工作page的时候用工作的密钥而不是私人page的：

在Mac上创建的所有密钥都会存到一个叫.ssh的文件夹里，.代表这个文件夹是隐藏的，在finder中是找不到的，需要指令打开：

open ~/.ssh

然后里面
就是你的密钥配置文件（config）还有密钥用户名 用户名后面带pub的是密钥密码。

看见两个文件： id_rsa id_rsa.pub 复制密钥 id_rsa.pub 到git账号里。

  • 新建user2的SSH Key

    #新建SSH key：
$ cd ~/.ssh 
ssh-keygen -t rsa -C "sheng0619@163.com"  # 新建工作的SSH key
# 设置名称为id_rsa_demsheng
Enter file in which to save the key (~/.ssh/id_rsa): id_rsa_demsheng

    新密钥添加到SSH agent中 因为默认只读取id_rsa，为了让SSH识别新的私钥，需将其添加到SSH agent中：

    ssh-add ~/.ssh/id_rsa_demsheng

    如果出现Could not open a connection to your authentication agent的错误，就试着用以下命令：

    ssh-agent bash
ssh-add ~/.ssh/id_rsa_work

  • 修改config文件

    在~/.ssh目录下找到config文件，如果没有就创建：

    touch config        # 创建config

    然后修改如下： 我的config配置如下：

    # 该文件用于配置私钥对应的服务器
# Default github user(first@mail.com)
Host github.com
HostName github.com
User git 
IdentityFile /home/lichangsheng/.ssh/id_rsa


    second user(second@mail.com)
    建一个github别名，新建的帐号使用这个别名做克隆和更新
    Host demsheng.github.com
    HostName github.com
    User git
    IdentityFile /home/lichangsheng/.ssh/id_rsa_demsheng

    其规则就是：从上至下读取config的内容，在每个Host下寻找对应的私钥。这里将GitHub SSH仓库地址中的git@github.com替换成新建的Host别名如：demsheng.github.com，那么原地址是：git@github.com:demsheng/sheng.git，替换后应该是：demsheng.github.com:funpeng/sheng.git.

  • 打开新生成的~/.ssh/id_rsa_demsheng.pub文件，将里面的内容添加到GitHub后台。

  • 以下命令可以检查当前用户： ssh -T git@github.com ssh -T git@demsheng.github.com

  • 以下命令可以检查当前秘钥： ssh-add -l

    • 下面总结一下git指令

    ssh-add命令是把专用密钥添加到ssh-agent的高速缓存中。该命令位置在/usr/bin/ssh-add。

    语法：
    ssh-add [-cDdLlXx] [-t life] [file …]
    ssh-add -s pkcs11

    选项
    -D：删除ssh-agent中的所有密钥.
    -d：从ssh-agent中的删除密钥
    -e pkcs11：删除PKCS#11共享库pkcs1提供的钥匙。
    -s pkcs11：添加PKCS#11共享库pkcs1提供的钥匙。
    -L：显示ssh-agent中的公钥
    -l：显示ssh-agent中的密钥
    -t life：对加载的密钥设置超时时间，超时ssh-agent将自动卸载密钥
    -X：对ssh-agent进行解锁
    -x：对ssh-agent进行加锁
    实例：
    1、把专用密钥添加到 ssh-agent 的高速缓存中：
    ssh-add ~/.ssh/id_dsa

    2、从ssh-agent中删除密钥：
    ssh-add -d ~/.ssh/id_xxx.pub

    3、查看ssh-agent中的密钥：
    ssh-add -l

    注：如果config不管用，重启一下电脑就好了。
